# Гайд по запуску прокси-сервера

**Что получим в итоге:**
- 🌐 Hysteria2 — быстрый прокси с защитой от DPI
- 🖥 h-ui — веб-панель для управления пользователями
- 📱 MTProxy — прокси для Telegram
- 🔒 Nginx + Let's Encrypt — HTTPS и заглушка-сайт
- 🛡 Fail2ban, UFW — защита сервера

**Что нужно заранее:**
- VPS с Ubuntu 22.04 — порты 80, 443 (TCP+UDP), 8443 открыты
- Свой домен с двумя поддоменами, например:
  - `vpn.твойдомен.com` — основной (заглушка-сайт)
  - `panel.твойдомен.com` — панель управления

---

## Шаг 1. Настройка DNS

В панели управления доменом добавить **две A-записи**:

| Тип | Имя | Значение | TTL |
|-----|-----|----------|-----|
| A | `vpn` | `IP_твоего_сервера` | 300 |
| A | `panel` | `IP_твоего_сервера` | 300 |

> ⏳ DNS обновляется до 30 минут. Проверить можно командой:
> `nslookup vpn.твойдомен.com` — должен вернуть IP сервера.

### Windows
```
Win + R → cmd → Enter

```

### Mac / Linux
```bash

```

## Шаг 3. Скачать проект с git

```bash
apt-get install -y git
git clone https://github.com/ВАШ_РЕПО/hystria2 /opt/proxy
cd /opt/proxy
```

---

## Шаг 4. Настройка конфига

На **сервере**:
```bash
cd /opt/proxy
cp .env.example .env
nano .env
```

Заполнить файл (стрелки для перемещения, `Ctrl+O` сохранить, `Ctrl+X` выйти):

```env
DOMAIN=vpn.твойдомен.com          # основной домен
HUI_SUBDOMAIN=panel.твойдомен.com # поддомен панели
EMAIL=твой@email.com               # для уведомлений Let's Encrypt
TZ=Europe/Moscow                   # часовой пояс
MTPROXY_PORT=8443                  # порт MTProxy (можно оставить)
SSH_PORT=2222                      # новый порт SSH (запомни!)
```

> ⚠️ Запомни `SSH_PORT` — после деплоя SSH будет работать на этом порту!

## Шаг 5. Запуск деплоя

```bash
cd /opt/proxy
chmod +x scripts/deploy.sh
sudo bash scripts/deploy.sh
```

Скрипт сделает всё автоматически (~5-10 минут):

```
[+] Устанавливаем Docker...
[+] Настраиваем фаервол...

[+] Настраиваем sysctl...

[+] Запускаем сервисы...
```

## Шаг 6. Настройка Hysteria2 в h-ui

### 6.1 Первый вход
Открой в браузере: `https://panel.твойдомен.com`

Логин: `admin` / Пароль: `sysadmin`

**Сразу смени пароль**: Настройки → Сменить пароль.

---

### 6.2 Настройка сервера

Перейди в **Настройки сервера** (Server Config) и заполни:

**Порт:** `443`

**TLS — сертификаты Let's Encrypt:**
```
Cert: /etc/letsencrypt/live/vpn.твойдомен.com/fullchain.pem
Key:  /etc/letsencrypt/live/vpn.твойдомен.com/privkey.pem
```

> h-ui видит эти файлы потому что они смонтированы в контейнер.

**Obfuscation (защита от DPI):**
```
Type:     salamander
Password: придумай_случайный_пароль  ← сгенерировать: openssl rand -base64 24
```
> Этот пароль — не для входа, а для маскировки трафика. Один на весь сервер,
> h-ui автоматически добавит его в ссылку подключения для клиентов.

**Masquerade (маскировка):**
```
Type: proxy
URL:  https://microsoft.com/
```

**Bandwidth:**
```
Up:   1 gbps
Down: 1 gbps
```

Нажать **Сохранить** → **Запустить сервер**.

Статус должен стать 🟢 Running.

---

### 6.3 Создание пользователя

Перейди в **Пользователи** → **Добавить**:

| Поле | Значение |
|------|----------|
| Имя | `мой_телефон` (любое) |
| Пароль | сгенерировать или ввести свой |
| Bandwidth up | 100 mbps (лимит для этого юзера) |
| Bandwidth down | 300 mbps |
| Статус | Активен |

После создания нажать на пользователя — h-ui покажет **ссылку для подключения** вида:
```
hysteria2://ПАРОЛЬ@vpn.твойдомен.com:443?...
```

## Шаг 8. Установка MTProxy

MTProxy устанавливается отдельно от Docker — как systemd сервис на хосте.

```bash
bash scripts/mtproxy.sh
```

Установщик задаст несколько вопросов:

| Вопрос | Ответ |
|--------|-------|
| Реализация | `2` — Official C (рекомендуется) |
| Порт | `8443` (или что задано в `.env`) |
| Секрет | Enter — сгенерирует автоматически |
| Tag | опционально, для статистики в @MTProxybot |

В конце установщик покажет ссылку вида:
```
tg://proxy?server=vpn.твойдомен.com&port=8443&secret=ee...
```

> ⚠️ Убедись что секрет начинается с `ee` — это fake-TLS режим, трафик
> маскируется под обычный TLS и не виден DPI.

**Добавить в Telegram:** открой ссылку в браузере — Telegram предложит добавить прокси автоматически.

## Управление сервером

### Базовые команды (выполнять на сервере в `/opt/proxy`)

```bash
# Статус всех контейнеров
docker compose ps

# Логи в реальном времени (Ctrl+C чтобы выйти)
docker compose logs -f

# Логи конкретного сервиса
docker compose logs -f h-ui
docker compose logs -f nginx

# Перезапустить всё
docker compose restart

# Остановить всё
docker compose down

# Запустить снова
docker compose up -d
```

### Если что-то сломалось

```bash
# Проверить что слушает на портах
ss -tulnp | grep -E '80|443|8443'

# Проверить сертификаты
docker compose run --rm certbot certificates

# Принудительно обновить сертификаты
docker compose run --rm certbot renew --force-renewal

# Перезапустить nginx
docker compose restart nginx

# Посмотреть кто заблокирован fail2ban
docker compose exec fail2ban fail2ban-client status
docker compose exec fail2ban fail2ban-client status nginx-4xx
```

### После перезагрузки сервера

Всё поднимается **автоматически** — systemd сервис `proxy-stack` запускает Docker Compose при старте системы. Ничего делать не нужно.

## Частые проблемы

**❌ Сертификат не выдаётся**
- Проверь что DNS указывает на IP сервера: `nslookup vpn.твойдомен.com`
- DNS мог ещё не обновиться — подождать и попробовать снова: `bash certbot/init.sh`

**❌ h-ui панель не открывается**
- Hysteria2 работает на том же порту 443 UDP — а панель идёт через nginx на 443 TCP. Это нормально, они не конфликтуют.
- Проверь: `docker compose ps` — все ли контейнеры `Up`?

**❌ Hysteria2 не подключается**
- Убедись что в клиенте указан тот же `obfs-password` что задан в h-ui
- Проверь статус сервера в h-ui — должен быть Running
- Открой UDP 443 в фаерволе: `ufw status` — должна быть строка `443/udp ALLOW`

**❌ Забыл SSH порт / заблокировал себя**
- В панели хостинга есть **VNC/Console** — доступ к серверу без SSH
- Войти через консоль, исправить порт: `nano /etc/ssh/sshd_config`

**❌ После деплоя SSH не работает на старом порту 22**
- Это нормально — порт сменился на тот что указан в `.env` (`SSH_PORT`)
- Подключаться: `ssh -p 2222 root@IP_сервера`
